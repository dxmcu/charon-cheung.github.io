(六) 前端 LocalTrajectoryBuilder2D
2020-08-03|激光SLAMCartographer源码解读|
Word count: 262|Reading time: 1 min

在Cartographer中机器人的运动轨迹是由一个个前后串联的节点(TrajectionNode)构成的。每当有新的扫描数据输入， LocalTrajectoryBuilder2D都会先通过扫描匹配器生成一个MatchingResult的对象，但是它的insertion_result字段是留空的。只有当通过运动滤波器成功插入到子图中之后，才会将插入的结果记录下来， 于此同时子图就得到了更新。

在前端部分有两个重要成员变量：

类RealTimeCorrelativeScanMatcher2D实现了子论文的算法，Match函数 Aligns ‘point_cloud’ within the ‘grid’ given an ‘initial_pose_estimate’ then updates ‘pose_estimate’ with the result and returns the score.

类CeresScanMatcher2D使用Ceres库在现有地图里align scans，Match函数 Aligns ‘point_cloud’ within the ‘grid’ given an ‘initial_pose_estimate’ and returns a ‘pose_estimate’ and the solver ‘summary’.
