slam_gmapping节点在gmapping包里面，它的节点信息如下：

它获取sensor_msgs/LaserScan的消息,然后用来建图。/slam_gmapping一边建图一边将地图数据发布到map话题，如果用rviz订阅map话题，我们就能看到建图的过程了.

slam需要的tf变换:

  • laser → base_link, 一般是固定的变换,使用static_transform_publisher节点实现
  • base_link → odom, 由里程计系统提供

slam_gmapping的参数

gmapping包生成三个节点,分别为slam_gmapping和slam_gmapping_replay及一个测试用的节点。我们主要看slam_gmapping节点,它用到了很多参数:

gmapping wrapper参数

  • throttle_scans: [int,默认1] throw away every nth laser scan
  • base_frame: [string, default:”base_link”] 机器人上用的坐标系frame_id
  • map_frame: [string,default:”map”] 机器人地图所用的frame_id
  • odom_frame: [string,default:”odom”] 里程计坐标系的frame_id
  • map_update_interval: [float,default:5.0] 两次更新地图的间隔,降低参数会更新occupancy grid更频繁,但增大CPU占用

Laser Parameters

  • maxRange [double] 雷达扫描的最大有效距离,超过这个数值的扫描会忽略,一般设置的比雷达真实最大扫描距离大一点

  • maxUrange [double,default: maxRange] 用于建图的最大可用扫描距离,一般设置的比雷达真实最大扫描距离小一点

  • sigma [float, default: 0.05] The sigma used by the greedy endpoint matching

  • kernelSize [int,default: 1] The kernel in which to look for a correspondence, search window for the scan matching process

  • lstep [float,default: 0.05] initial search step for scan matching (linear)

  • astep [float,default: 0.05] initial search step for scan matching (angular)

  • iterations [int, default: 5] number of refinement steps in the scan matching. The final “precision for the match is lstep2^(-iterations) or astep2^(-iterations), respectively.

  • lsigma [float, default: 0.075] The sigma of a beam used for likelihood computation

  • ogain [float, default: 3.0] Gain to be used while evaluating the likelihood, for smoothing the resampling effects

  • lskip [int, default: 0] take only every (n+1)th laser ray for computing a match (0 = take all rays) 表示所有的激光都处理，尽可能为零，如果计算压力过大，可以改成1

  • minimumScore [float, default: 0.0] minimum score for considering the outcome of the scanmatching good. Can avoid ‘jumping’ pose estimates in large open spaces when using laser scanners with limited range (e.g. 5m). (0 = default. Scores go up to 600+, try 50 for example when experiencing ‘jumping’ estimate issues) 最小匹配得分，这个参数很重要，它决定了你对激光的一个置信度，越高说明你对激光匹配算法的要求越高，激光的匹配也越容易失败而转去使用里程计数据，而设的太低又会使地图中出现大量噪声，所以需要好好调整。

Others:

  • linearUpdate: [float,默认1] 机器人走过多少米,gmapping才处理雷达scan数据

  • angularUpdate: [float,默认0.5] 机器人转过多少弧度,gmapping才处理雷达scan数据

  • temporalUpdate: [float,默认-1] 两次雷达scan数据之前的等待时间,小于0则不启用这个功能

  • resampleThreshold [float,default: 0.5] 粒子重新采样的门槛值,越大表示重新采样越频繁

  • particles: [int,默认30] 决定gmapping算法中的粒子数，每个例子代表了机器人走过的可能的trajectory,因为gmapping使用的是粒子滤波算法，粒子在不断地迭代更新，所以选取一个合适的粒子数可以让算法在保证比较准确的同时有较高的速度

运动模型参数 (高斯噪声模型的所有标准偏差)

  • srr [float, default: 0.1] linear noise component (x and y),Odometry error in translation as a function of translation (rho/rho)

  • srt [float, default: 0.2] linear -> angular noise component,Odometry error in translation as a function of rotation (rho/theta)

  • str [float, default: 0.1] angular -> linear noise component,Odometry error in rotation as a function of translation (theta/rho)

  • stt [float, default: 0.2] angular noise component (theta),Odometry error in rotation as a function of rotation (theta/theta)

Likelihood sampling (used in scan matching)

  • llsamplerange [float, default: 0.01] Translational sampling range for the likelihood
  • lasamplerange [float, default: 0.01] Angular sampling range for the likelihood
  • llsamplestep [float, default: 0.005] Translational sampling step for the likelihood
  • lasamplestep [float, default: 0.005] Angular sampling step for the likelihood

地图初始维度和分辨率

  • xmin [double] minimum x position in the map [m]
  • ymin [double] minimum y position in the map [m]
  • xmax [double] maximum x position in the map [m]
  • ymax [double] maximum y position in the map [m]
  • delta [double] 地图分辨率,就是话题map_metadata中的resolution
