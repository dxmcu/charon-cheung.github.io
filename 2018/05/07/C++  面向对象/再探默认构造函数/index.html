这个问题的实质还是默认构造函数。我们都知道，如果一个类没有构造函数，编译器会为我们自动创建一个默认构造函数，但这个函数什么都没做。但是，当我们实现一个构造函数之后，编译器就不会创建。
C++中，最多有一个默认构造函数，或者是编译器生成，或者是我们自己定义的。我们自己定义的默认构造函数只能是两种：1. 无参的构造函数 2. 有参数，但参数都有默认值的构造函数。 比如下面两种都是默认构造函数：

如果某派生类的基类没有默认构造函数，那么编译器也不会为该派生类隐式地定义默认构造函数，就会出现上面的错误。对这种错误，有两种修改方法：
