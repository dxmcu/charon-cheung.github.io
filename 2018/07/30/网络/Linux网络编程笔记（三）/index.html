socket函数

linux上可以在socket()函数创建socket时指定创建的socket是异步的，在type的参数中设置SOCK_NONBLOCK标志即可，例如：int s = socket(AF_INET, SOCK_STREAM | SOCK_NONBLOCK, IPPROTO_TCP);

windows和linux平台上accept()函数返回的socekt也是阻塞的，linux另外提供了一个accept4()函数，可以直接将返回的socket设置为非阻塞模式：int accept4(int sockfd, struct sockaddr *addr, socklen_t *addrlen, int flags);只要将accept4()最后一个参数flags设置成SOCK_NONBLOCK即可。

将socket设置成非阻塞模式，还可以通过API函数fcntl()来设置：fcntl(sockfd, F_SETFL, fcntl(sockfd, F_GETFL, 0) | O_NONBLOCK);

端口立即重用，setsockopt函数

socket编程时，常遇到下面的错误：Cann't bind server socket ! Address already in use。在bind的时候，端口被占用，更常见的情况是同一个程序运行结束后，网络处于TIME_WAIT状态，需要等待2MSL的时间，套接字被删除，该地址才能被重新绑定而不出问题。

setsockopt常用于端口立即重用，在socket函数之后，bind函数之前。在绑定地址之前，以SO_REUSEADDR选项调用setsockopt，为了允许地址重用，设置整型参数（on）为 1 （否则可以设为 0 来禁止地址重用）。

fd是socket函数返回的文件描述符，第二个参数表示级别，一般有SOL_SOCKET，IPPROTO_IP，opt用于赋值。
