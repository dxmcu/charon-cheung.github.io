MySQL配置文件

MySQL主要配置文件：my.ini(windows)或者/etc/my.cnf(Linux)二进制日志，错误日志，查询日志，数据文件。

二进制日志：用于主从复制，在my.ini中的[mysqld]的log-bin参数。
错误日志：默认关闭，记录严重警告和错误信息，每次启动和关系的信息。在[mysqld]的log-error
查询日志log：默认关闭，记录查询的sql，如果开启会降低mysql性能，与慢查询日志有关。

数据文件：windows的F:\mysql-5.6.24-win32\data存放数据库的文件，Linux的/var/lib/mysql存放数据库。在数据库的目录例如data\mysql里，存放三种文件frm,myd,myi，分别存放表结构、表数据、表索引。

Workbench自动生成sql语句出错

通过MySQL workbench生成的sql语句可能会有错。
unknown database ‘Robot‘，这是给Robot加了符号，需要在sql文件里手动去掉，也就是不加符号。 报错variable ‘sql_mode’ can’t be set to the value of ‘NULL’` ，去掉不必要的注释即可。

Incorrect table definition; there can be only one auto column and it must be defined as a key

一个表只能有一个自增列，该列必须是主键，自增的属性列必须定义为数值（int等）类型。
