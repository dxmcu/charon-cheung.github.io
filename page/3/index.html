PL-ICP算法 2020-04-08|激光SLAM

里程计标定精度到0.3%算很好了

里程计短时精度高，但会累积误差。因此经常使用里程计确定两帧间的位姿差，用于slam帧间优化中的初始值。但是激光匹配得到的位姿差和里程计得到的位姿不一定能完全对应上，这里就是通过标定减少这种差异

从题目本身来说吧，你可以看看ppt P23有一句，用激光scan match的数据作为真值。而scan match计算的位姿就是以上一帧为参考的，你可以结合所有代码研究下

里程计由于是编码器计算输出的，输出的角速度也有可能和实际的角速度差个0.几倍，这部分的误差标定就相当于是放到轮子内参一起标了

线性关系当然有可能是不成立的。线性标定和基于模型标定相当于一个是黑盒标定，一个是白盒标定。有些传感器当不知道它具体模型的时候，或者说无法用第二种方法进行标定的时候，假定它是一个线性关系，也不妨是一种办法

这里用激光的匹配结果来作为groundtruth来对里程计进行标定，是因为激光匹配的精度相比里程计会更高一些。实际标定的时候也可以选用更精准的groundtruth真值，比如人手工高精度测量，或者室外配备高精度惯导设备

做帧间的匹配是为了得到机器人前后的局部位姿变换关系，可以认为是一种里程计。其实标定了轮子直径跟间距我们就有了一个里程计了，但是因为轮子打滑等各种因素存在误差会比较大。再用雷达的数据做一次配准可以提高定位精度。

使用Gmapping建图需要提供Odometry和laser，但是只用一个激光雷达也可以进行Gmapping建图，用于没有轮式里程计的场合，比如手持设备，背包，无人机．采用的方法就是使用 laser_scan_matcher 功能包。该软件包允许扫描连续的 sensor_msgs / LaserScan 消息之间的匹配，并将估计的激光位置发布为 geometry_msgs / Pose2D 或 tf 变换。该包可以在没有其他传感器提供的任何测距估计的情况下使用。因此，scan match被称为激光里程计．

PLICP是二阶收敛，ICP是一阶收敛
PLICP对初始值更敏感
以点到线的误差为目标函数，PLICP的求解精度更高

pf_init 函数 2020-04-04|激光SLAMamcl和粒子滤波

原型：void pf_init(pf_t *pf, pf_vector_t mean, pf_matrix_t cov)

pf是之前初始化的粒子滤波器对象，mean是amcl中的初始位姿[x,y,θ], cov是初始协方差

调用 pf_cluster_stats的函数


源码里没有用到pf_init_model，它完成的功能与pf_init基本是一样的，只是pf_init使用的是高斯模型，pf_init_model可以由用户提供模型。用户可以定义一个service客户端，发起global_localization服务，AMCL源码已经定义好了回调函数。

pf_init_model有三个参数，其中pf仍然是滤波器对象。init_fn是一个函数指针，用户需要提供一个生成随机样本的函数实现，该参数的作用就相当于pf_init中的高斯分布对象。参数init_data则是生成粒子的样本空间， 我们可以为之赋予地图数据。
